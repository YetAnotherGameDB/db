# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a Game Meta Database — a JSON-based database that links game entries to external database sources (TheGamesDB, GameEye, PriceCharting, Redump). The primary data file is `games.json`.

## Validation

Run locally with PowerShell:
```powershell
# Validate JSON structure and unique IDs
pwsh .github/scripts/validate-games.ps1

# JSON lint (requires jsonlint installed)
jsonlint -q games.json
```

CI runs both checks automatically on every push via GitHub Actions (`.github/workflows/validation.yml`).

## Data Structure

### `games.json`
The main database. Top-level structure:
```json
{
  "enums": {
    "platforms": ["ps1", "saturn", "snes"],
    "regions": ["NTSC-J", "NTSC-U", "PAL"]
  },
  "games": [ ... ]
}
```

Each game entry:
```json
{
  "id": 1,
  "name": "GAME TITLE",
  "releaseDate": "1994-12-03",
  "gameInstances": [
    {
      "name": "...",
      "regionalName": "...",
      "platform": "ps1",
      "language": "[J]",
      "serial": "SLPS-XXXXX",
      "region": "NTSC-J",
      "tgdbId": 0,
      "GameyeID": 0,
      "PriceCharting": "jp-playstation/game-slug",
      "UPC": "..."
    }
  ]
}
```

- `id` must be a unique integer across all entries — enforced by the validation script.
- A game can have multiple `gameInstances` for different regional releases.
- `tgdbId: 0` means the TheGamesDB ID has not been looked up yet.

### `psx_games.json`
A separate flat list generated by `redump_extractor.ps1`. Contains Redump DAT data (md5, sha1, serial) for PSX games. IDs are prefixed: `"psx-1"`, `"psx-2"`, etc.

## Utility Scripts

- **`injectid.ps1`** — Reassigns sequential integer IDs to all entries in `games.json`. Run after manually reordering entries.
- **`redump_extractor.ps1`** — Downloads a Redump DAT file for a given platform and converts it to JSON. Usage:
  ```powershell
  .\redump_extractor.ps1 -Platform psx
  ```
  Skips games already present (deduplicates by MD5). Outputs to `{platform}_games.json`.

## Key Conventions

- Game names in `games.json` are uppercase.
- After adding or reordering games in `games.json`, run `injectid.ps1` to regenerate IDs, then validate.
- Do not manually edit IDs if you are adding many entries — use `injectid.ps1` instead.
- `psx_games.json` is separate from `games.json` and managed by `redump_extractor.ps1`.
